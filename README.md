# 艾特智能写作工厂 (Aito Intelligent Writing Factory) 🚀

艾特智能写作工厂是一款专为史诗长篇小说创作而设计的**全自动、多智能体深度协同的 AI 写作与管理平台**。平台基于大语言模型（LLM）驱动，通过创新的**多幕场景迭代扩写**与**世界观/角色记忆数据库管理**机制，能够一键将几百字的核心故事大纲，转化为结构严谨、细节丰满、风格统一的十万字乃至百万字的长篇大作。

---

## 🌟 核心特性

- **大纲切分与任务编排**：系统能够自动分析用户输入的核心小说大纲，并将其切分为结构清晰的章节任务队列（Chapter Task Queue），科学规划每章的卷名、冲突、出场角色与伏笔。
- **三阶段深度场景扩写**：摒弃传统 AI 一次性生成数千字的扁平化写法。每一章都严格分拆为 `环境引入与氛围铺垫`、`核心冲突与多轮对话`、`冲突收尾与高潮余韵` 三幕独立撰写，并在必要时自动执行深度扩写循环，确保字数达标且细节详实。
- **智能文风与笔触模仿**：支持导入期望的文风样板段落（如：古龙风格、金庸风格、张爱玲风格、鲁迅风格等），让 AI 在写作过程中深度对齐样板的语调、句式结构和叙事张力。
- **世界观记忆数据库**：在写作过程中实时提取并维护`核心角色动态`、`关键道具去向`、`情感关系线走向`和`剧情悬念状态`，并自动作为上下文输入下一章，彻底杜绝长文本创作中角色设定崩塌或“吃设定”的问题。
- **实时执行管线监控**：提供极具科技感的可视化看板。包含章节消费格子图、实时执行日志终端、手稿分段折叠预览器（支持一键收起/展开，只展示一二级标题大纲），以及记忆库动态查看卡片。

## 🛠️ 本地环境部署与运行指南

### 1. 基础环境要求
- **操作系统**：Linux / macOS / Windows
- **运行环境**：Python 3.8 及以上版本

### 2. 安装依赖库
在项目根目录下，使用 `pip` 安装所需的全部核心依赖：
```bash
pip install -r requirements.txt
```

### 3. 配置环境变量
在项目根目录下创建一个 `.env` 配置文件，并配置您的 LLM API 密钥和相关参数(我们以接入deepseek为例)：

```
cp .env.example .env
```

```ini
# 默认/首选模型配置 (不带任何引号，支持用英文逗号分隔配置多个模型)
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_API_BASE=https://api.deepseek.com
OPENAI_MODEL_NAME=deepseek-chat,deepseek-coder

# 支持配置多个额外模型，供 Web 界面选择 (可选，亦支持英文逗号分隔)
MODEL_1_NAME=gpt-4o,gpt-4-turbo,gpt-3.5-turbo
MODEL_1_API_KEY=sk-xxxxxx
MODEL_1_API_BASE=https://api.openai.com/v1

MODEL_2_NAME=claude-3-5-sonnet
MODEL_2_API_KEY=sk-yyyyyy
MODEL_2_API_BASE=https://api.anthropic.com/v1
```

> [!WARNING]
> 在配置 `.env` 文件时，请勿在键值两侧加任何单引号或双引号（例如用 `OPENAI_API_KEY=sk-...` 代替 `OPENAI_API_KEY="sk-..."`）。Docker 容器在加载环境参数时，会将引号本身作为 API 密钥的一部分读取，从而导致 401 认证失败。


### 4. 本地源码运行
执行以下命令启动 Web 管理平台：
```bash
uvicorn app:app --host 127.0.0.1 --port 8000 --reload
```
启动成功后，在浏览器中访问：`http://127.0.0.1:8000` 即可进入控制台。

## 🛠️ Docker 容器化部署 (推荐)

本项目支持一键进行 Docker 部署。通过挂载 `projects` 目录，可以确保容器重启后生成的小说手稿与记忆库不丢失。

### 1. 构建镜像
在项目根目录下执行以下命令构建 Docker 镜像：
```bash
docker build -t aitobox/atb-novel:latest .
```

### 2. 运行容器
1.  `cp .env.example .env`， 填入自己的KEY

2. 使用以下命令启动 Docker 容器：
```bash
docker run -d \
  -p 8000:8000 \
  --name atb-novel-factory \
  -v $(pwd)/projects:/app/projects \
  --env-file .env \
  aitobox/atb-novel:latest
```

*说明：*
- `-p 8000:8000`：将主机的 8000 端口映射到容器的 8000 端口。
- `-v $(pwd)/projects:/app/projects`：将宿主机当前目录下的 `projects/` 文件夹挂载到容器内，保证小说项目数据持久化。
- `--env-file .env`：直接读取宿主机上的环境变量配置文件（包含您的 API 秘钥配置）。

## 📖 平台使用指南

### 第一步：新建小说项目与设定
1. 点击左上角的 **“新建”** 按钮。
2. **填写基本信息**：
   - 输入**小说名称**。
   - 设定**流派风格与风格笔触**（可直接在下方选择快速标签，如 `末世/诡异/单女主`，多标签以 `/` 分割）。
3. **规划字数与章节**：
   - 拖动滑块设置规划章节总数（5 - 100 章）与单章最低字数指标（1000 - 5000 字）。
4. **填入核心大纲**：
   - 在文本框中详细输入小说的故事大纲或背景设定。
5. **设定期望风格样板**：
   - 在下拉菜单中选择一个经典的作家文风模板（如**古龙风格**、**金庸风格**等），或者在此粘贴任意您自己期望模仿的短文片段。
6. 点击 **“一键开启自动化管线”**。

### 第二步：管线监控与章节生成
- 进入项目详情页，您会看到：
  - **章节编排大纲与任务消费状态**：每个方格代表一个章节任务，悬浮即可查看本章大纲摘要与伏笔要求。灰色代表等待，紫色闪烁代表正在创作，绿色代表已完成。
  - **实时执行管线监控终端**：实时输出 AI 创作时的调用日志与运行状态。
- 点击 **“启动管线”**（或暂停），即可开始自动连载生成。

### 第三步：最新手稿预览与记忆库
- **手稿预览**：实时预览已生成的整部小说。
  - 点击每一章折叠卡片，可查看具体正文。
  - 支持 **“展开全部章节”** 或 **“收起全部章节”**。点击“收起”后，正文会完全隐藏，仅保留一二级标题的清晰大纲视图。
- **世界观记忆数据库**：在右下角实时展示提取自已生成章节的角色状态、最新位置、物品所有权变动以及剧情关系。
- **导出文稿**：点击右上角 **“导出小说文稿”** 按钮，可一键下载格式标准的 Markdown 版小说手稿（`master_novel.md`）。

### 3. 生成提示词 (Prompts) 原理
当后台调用 `NovelGenerator` 创作《神座之上：欺诈者传奇》时，程序会拼装以下提示词给 LLM：
- **前情提要提示词（以撰写第 19 章为例）**：
  > 汇聚最近完成的章节摘要，如：`第 18 章《血色王座上的演说》剧情摘要：李星赫在审判庭公开指证塞拉斯伪造证据...` 自动作为 `{recent_context}` 输入。
- **环境氛围渲染（第一幕）**：
  > “你是一位文笔细腻、擅长铺陈描写的小说家。你正在写一部史诗长篇网文《神座之上：欺诈者传奇》，风格属于：游戏/无敌/进化/无限。当前写作章节：第 19 章《骑士的抉择》。现在你只需撰写【第一部分：场景引入与氛围铺垫】...
